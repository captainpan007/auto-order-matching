# 超市采购对账自动化系统

## 项目概况

校内超市（约20000名学生）采购对账自动化项目。
通过OCR识别供应商送货单PDF + 比对用友系统Excel，自动完成对账。

- **当前成绩**：采购对账 88.9%，财务对账 72.5%（10家供应商）
- **技术栈**：Python + 腾讯云OCR(RecognizeTableAccurateOCR) + openpyxl + PyMuPDF + Flask
- **运行环境**：Windows，Anaconda Python

## 对账流程

### 阶段一：采购对账（送货单 vs 采购入库单）

匹配策略（4级递进 + 行内搜索）：
1. PO号 + 条码 + 仓库
2. PO号 + 条码
3. 条码 + 仓库
4. 仅条码
5. 行内搜索（OCR行所有单元格中搜条码+金额）
6. 模糊条码（12位补位/14位截取/编辑距离=1）

一致性判定：
- 数量+单价+金额全一致 → 一致
- 金额近似（差≤2%且≤1.5元）→ 一致
- 数量一致+单价差≤0.1 → 一致
- 金额一致但数量/单价不同 → 一致（单位换算）
- 条码匹配但跨PO数量不同 → 一致（跨PO差异）
- 以上都不满足 → 差异

### 阶段二：财务对账（进货单 vs 送货单 vs 退货单）

逻辑同阶段一。退货单独列出，应付 = 进货 - 退货。

## OCR配置

- **API**：腾讯云 RecognizeTableAccurateOCR（统一接口，不再用GeneralAccurateOCR）
- **V3模式**：炬博、优亿家使用 UseNewModel=true
- **太古**：可口可乐专用解析器（_extract_taigu_items）
- **缓存**：_ocr_cache.json / _ocr_cache_v3.json，避免重复API调用

## 供应商特殊处理

| 供应商 | 送货单格式 | 特殊处理 |
|--------|----------|---------|
| 承运 | 条码+品名合并/分开 | √勾选符号去除 |
| 优品味 | 序号\|条码\|品名\|零数\|单价 | P0→PO格式修正 |
| 优亿家 | 商品名称\|条形码\|数量\|价格 | V3模式+行内搜索补全 |
| 翔瑞 | 序号\|条形码\|订单数量\|销售价 | 手写√标记 |
| 锐仕 | 序号\|商品条码\|散数\|散价\|金额 | 列名换行拆分 |
| 炬博 | 商品条码\|整件数\|单价\|单支总数 | V3模式+无表头推断+跨PO |
| 鑫凯 | 商品编码\|条形码\|数量\|单价 | 多表格长页 |
| 通力 | 类似承运 | 排除规格X误判缺货 |
| 太古 | 可口可乐特殊多表格 | 专用解析器+金额近似 |
| 汇基 | 类似承运 | 有退货单 |

## 技术约束

- 用友系统封闭，只能导出Excel
- 供应商送货单为纸质扫描PDF，需OCR
- 数据不出本地（OCR接口除外）
- 后续还有18家供应商待接入

## 成功标准

- 出错率 < 千分之五
- 采购助理核对时间减少70%以上
- 正常单据零人工干预，只处理差异项
